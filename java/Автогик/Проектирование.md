Проектирование - создание фундамента на котором будет стоять код и бизнес-логика.

==Исправление ошибки на этом этапе будет стоить в десятки раз дешевле, чем исправление на этапе кода.==

Этапы проектирования
1. Исследование
	1. [[Автогик/1 Исследование/Анализ рынка и конкурентов|Анализ рынка и конкурентов]] - изучение аналогичных решений
	2. [[Автогик/1 Исследование/Целевой клиент|Создание картины целевого клиента]]
	3. [[Автогик/1 Исследование/Целевой клиент|Картина решения проблемы клиента]] - визуализация пути пользователя от возникновения проблемы до её решения
2. Сбор и анализ требований
	1. [[Автогик/2 Сбор требований/Функциональные требования|Функциональные требования]] - что продукт должен делать
	2. [[Автогик/2 Сбор требований/Нефункциональные требования|Нефункциональные требования]] - как продукт должен работать (безопасность, нагрузка, скорость отклика)
	3. [[Автогик/2 Сбор требований/User Stories|User Stories]] - короткие описания функций в формате «Я, как пользователь, хочу \[действие], чтобы \[профит]».
3. Информационная архитектура - структура данных и связь их друг с другом
	1. Карта сайта/приложения - иерархия разделов
	2. Логические схемы (Flowcharts) - описание переходов между экранами и условий
4. UX-проектирование (Wireframing) - создание макетов продукта без дизайна
	1. Low-fidelity - наброски логики
	2. Тестирование гипотез - проверка интуитивной понятности интерфейса
5. UI-дизайн (Visual Design) - создание комфортной визуализации
	1. Design System - набор компонентов, чтобы продукт выглядел единообразно
	2. High-fidelity прототипы - макет готового приложения
6. Техническое проектирование
	1. Выбор стека технологий - языки программирования, БД, фреймворки
	2. Проектирование БД - схемы таблиц и связей
	3. API -  описание взаимодействия фронтенда с бекендом
