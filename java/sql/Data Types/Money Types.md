тип ==money== хранит денежную сумму с фиксированной дробной частью в диапазоне от -92233720368547​758.08 до +92233720368547​758.07

==объем данных 1 байт==

точность дробной части определяется на уровне базы данных параметром lc_monetary

входные данные могут быть записаны:
 - в виде целых и дробных чисел
 - в виде строки в денежном формате
	например '$1,000.00'

выводятся эти значения обычно в денежном формате, зависящем от региональных стандартов.

т.к. выводимые значения этого типа зависят от региональных стандартов, попытка загрузить данные в базу данных с другим параметром lc_monetary может быть неудачной

значения типов [[sql/Data Types/Numeric/числа с точкой/numeric|numeric]], [[sql/Data Types/Numeric/целые числа/int|int]] и [[sql/Data Types/Numeric/целые числа/bigint|bigint]] можно привести к типу money
преобразования типов [[sql/Data Types/Numeric/числа с точкой/real|real]] и [[sql/Data Types/Numeric/числа с точкой/double precision|double precision]] производятся через тип numeric
 SELECT '12.34'::float8::numeric::money;

Значение money приводится к типу numeric без потери точности
преобразование в другие типы может быть неточным и также должно выполняться в два этапа
 SELECT '52093.89'::money::numeric::float8;

при делении значения типа money на целое число выполняется отбрасывание дробной части и получается целое, ближайшее к нулю
для получения результата с округлением, выполняется деление значения с плавающей точкой или приведение значения типа money к numeric до деления, затем приведение результата к типу money
(Последний вариант предпочтительнее, так как исключает риск потери точности)

когда значение money делится на другое значение money, результатом будет значение типа double precision
денежные единицы измерения при делении сокращаются.
