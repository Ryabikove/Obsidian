хранит числа с фиксированной точностью. Данный тип может принимает два параметра precision и scale: DECIMAL(precision, scale)

==без указания precision и scale создается столбец "неограниченных чисел", где можно хранить любые значения до предела типа
в столбце этого типа входные значения не будут приводиться к какому-либо масштабу==

==precision==
максимальное количество цифр, которые может хранить число, включая числа за точкой
значение должно находиться в диапазоне от 1 до 1000

==scale==
количество цифр, которые может содержать число после запятой. 
значение должно находиться в диапазоне от 0 до precision
по умолчанию 0

числовые значения физически хранятся без каких-либо дополняющих нулей слева или справа
объявляемые точность и масштаб столбца определяют максимальный, а не фиксированный размер хранения

==количество цифр слева от точки определяется по принципу scale - precision, и никак не может быть больше этого значения==


==объем данных зависит от хранимого значения==

==математические операции выполняются медленнее, чем с целочисленными значениями или числами с плавающей точкой==

если scale значения, превышает объявленный scale столбца, система округлит его до заданного количества цифр после точки
если же после этого количество цифр слева в сумме с scale превысит precision, произойдёт ошибка

==может не поддерживаться другими бд==
PostgreSQL позволяет использовать в объявлении типа decimal любое значение scale в диапазоне от -1000 до 1000

==может не поддерживаться другими бд==
начиная с PostgreSQL 15, столбец типа numeric и decimal можно объявить с отрицательным scale
в этом случае значения будут округляться до позиции слева от десятичной точки
при этом precision так же задаёт максимальное число цифр, без учёта округления
 decimal(2,-1) = от -990 до 990 включительно

==может не поддерживаться другими бд==
также вполне возможно задать scale, превышающий precision
такой столбец сможет содержать только дробные значения, в которых количество нулей сразу после точки должно равняться разности scale и precision
 decimal(3,6) = от -0.000999 до 0.000999 включительно

так же decimal принимает специальные значения:
 - Infinity
 - -Infinity
 - NaN

==при округлении выдает число, большее по модулю==
