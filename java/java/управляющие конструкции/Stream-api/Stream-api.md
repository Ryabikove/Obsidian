Набор инструментов, предназначенный для удобной эффективной обработки последовательностей элементов в функциональном стиле.

Ключевая концепция - stream - конвейер операций над элементами, взятыми из источника. Источником может быть:
- [[обучение/Java core/Коллекции|коллекция]]
- [[java/переменные/атрибуты переменных/значения переменной/Массив|Массив]]
- файл
- генератор значений и д.р.

Способы вызова последовательных потоков:
- Для коллекций через `stream()` метод - `array.stream().filter(p -> p.length() > 5).toList();`
- Для массивов через `stream()` метод класса-коллекции - `Arrays.stream(arr).filter(p -> p.length() > 5).toList();`

Основные св-ва:
- Данные принимаются из источника и после обработки передаются сразу дальше, а не хранятся в памяти
- Операции возвращают новый поток или результат, но не модифицируют исходные данные
- Промежуточные операции не выполняются сразу. Реальные вычисления запускаются только при вызове терминальной операции.
- Потоки могут быть последовательными или [[java/управляющие конструкции/Stream-api/Параллелизация|параллельными]]

Операции в потоке делятся на промежуточные (intermediate) и терминальные (terminal).

Промежуточные операции преобразуют поток и возвращают новый поток. Могут выстраиваться в цепочку. К таким операциям относятся:
- filter([[java/пакет/встроенные/java.util/function/Predicate|Predicate]]) - отбирают элементы по условию;
- map([[java/пакет/встроенные/java.util/function/Function|Function]]) - выстраивает список эл-тов в один поток и преобразуют каждый;
- flatMap([[java/пакет/встроенные/java.util/function/Function|Function]]) - то же, что и map(), только может разделять многомерные массивы на эл-ты;
- distinct() - удаляет дубликаты;
- sorted() - сортирует элементы;
- limit(n) - ограничивает количество элементов;
- skip(n) - пропускает первые n элементов;

Терминальные операции завершают работу с потоком и возвращают результат, после их выполнения поток становится недоступным. К таким операциям относятся:
- forEach(action) - выполняет действие для каждого элемента;
- collect([[java/пакет/встроенные/java.util/stream/Collector|Collector]]) - собирает эл-ты в коллекцию;
- reduce(accumutalor) - сводит эл-ты к одному значению;
- count() - возвращает кол-во элементов;
- findFirst() - возвращает первый эл-т;
- anyMatch([[java/пакет/встроенные/java.util/function/Predicate|Predicate]]) - проверят, есть ли эл-ты, удовлетворяющие условию;
- toList() - возвращает эл-ты в виде коллекции List;

Основные ==преимущества==:
- Функциональный стиль - поддержка [[java/управляющие конструкции/Лямбда-выражения|лямбда-выражений]] и [[java/пакет/класс/метод/Ссылки на методы|ссылок на методы]];
- Параллелизация - возможность переключения на [[java/управляющие конструкции/Stream-api/Параллелизация|параллельную]] обработку;
- Гибкость - можно комбинировать операции в цепочки для сложной обработки данных;

==Недостатки==:
- Одноразовость - после терминальной операции нельзя использовать повторно;
- Ограниченная область применения - не все задачи удобно решать с помощью потоков;
