Лямбда-выражения - компактный способ записи анонимных функций.
В основе выражения лежит понятие "[[java/пакет/класс/наследование/Функциональный интерфейс|Функциональный интерфейс]]".
Компилятор сам догадывается, какой метод вы реализуете, основываясь на типе переменной.

Синтаксис: 
`(параметры) -> { тело выражения }`

Существует несколько стандартных способов записи выражения:
```
Полный: (String s) -> { return s.length(); } # По сути анонимная функция

Без типов: (s) -> { return s.length(); } # Java сама понимает тип из контекста

Без скобок: s -> { return s.length(); } # Если параметр один, круглые скобки не нужны.

Минимализм: s -> s.length() # Если в теле одно выражение, можно убрать {} и return.
```

Помимо стандартных способов записи можно использовать [[java/пакет/класс/метод/Ссылки на методы|ссылки на методы]]:
`Class::method`, это сокращает запись и делает код чище.
Однако такой способ записи не всегда возможен.


==Нюансы==:
- Если параметров нет или более двух, скобки обязательны: `() -> System.out.println("Hi")`
- Если используются фигурные скобки, то return (если метод что-то возвращает) обязателен
- Выражение не может существовать само по себе, ему необходим контекст. Для этого служат [[java/пакет/класс/наследование/Функциональный интерфейс|функциональные интерфейсы]] и [[java/управляющие конструкции/условные/switch|switch конструкция]].


Лямбда выражения встречаются в основном:
- При присваивании переменных, чтобы использовать логику позже:
```
Predicate<String> isNotEmpty = s -> !s.isEmpty();
boolean result = isNotEmpty.test("Hello"); // true
```
- В качестве аргумента метода - т.к. некоторые методы ждут "действия", например, метод `removeIf` у списков ждет `Predicate` (условие для удаления). `names.removeIf(s -> s.isEmpty());`
- В качестве возвращаемого значения: метод иногда возвращает логику, а не просто значение. Полезно, когда необходимо использовать одну и ту же логику несколько раз в коде. В отличие от переменной того же типа, может подставлять свои значения.
```
public Predicate<String> getPasswordValidator(int minLength) {
	return s -> s.length() > minLength;
}
```
 - Внутри конструкции [[java/управляющие конструкции/условные/switch|switch]]


Компилятор определяет тип лямбды с помощью процесса **Target Typing**:
Если метод описан как `void doWork(Runnable r)`, и ты вызываешь его `doWork(() -> System.out.println("Done"))`, компилятор рассуждает так:
1. Метод `doWork` ждет объект типа `Runnable`.
2. В интерфейсе `Runnable` есть один метод `run()`, который ничего не принимает и ничего не возвращает.
3. Твоя лямбда `() -> ...` тоже ничего не принимает и не возвращает.
4. **Матч!** Значит, эта лямбда и есть реализация `run()`.

Термин **Target Typing** (вывод целевого типа) — это официальное понятие из **Java Language Specification (JLS)**, а именно из главы 15. Это фундаментальный механизм, на котором держится работа лямбд в Java.
