набор объединенных логически классов.
для указания того, что класс принадлежит пакету используется ключевое слово package.
 package \*название пакета\*;

==требования==:
 - указание принадлежности к пакету должно писаться первой непустой строкой в файле
 - при компиляции программы в терминале все действия необходимо производить из родительского каталога пакета

==особенности==:
 - объединение классов в виде пакета позволяет избежать конфликта имен между классами, т.к. одинаковые названия классов могут быть в разных пакетах
 - названия пакетов, как правило, соответствуют организации каталогов проекта, в которых находятся файлы с исходным кодом
 - если пакет не определен для класса, то считается, что класс находится в  пакете без имени
