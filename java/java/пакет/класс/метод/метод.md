набор операторов класса, выполняющих определенные действия
простыми словами - функция, содержащаяся в конкретном классе и вызывающаяся из этого класса

в состав метода входят:
 - [[пакет/класс/модификаторы/модификатор|модификаторы]]
 - [[переменные/атрибуты переменных/типы данных/тип данных|тип возвращаемого значения]]
 - [[переменные/атрибуты переменных/имя объекта]]
 - [[пакет/класс/метод/передаваемые параметры|передаваемые параметры]]
 - тело метода - [[блок кода]], идущий после строки определения метода

синтаксис определения метода:
```
[модификаторы] тип_возвращаемого_значения имя_метода ([параметры]) {
	 блок кода
 }
```

==требования==:
 - рекурсивная функция обязательно должна иметь оператор return, помещенный вначале функции, и вызываемый по определенному условию, иначе  произойдет бесконечная вложенность
 - метод всегда должен иметь перед названием указатель на тип возвращаемого значения
	`Type method(){}`
 - абстрактный метод должен быть переопределен в любом из наследуемых классов, но необязательно в непосредственно наследуемом

==особенности==:
 - метод помимо принятия параметров может их так же возвращать с помощью оператора-ключевого слова [[пакет/класс/метод/return|return]]
 - можно использовать несколько методов с одинаковыми именами
	[[пакет/класс/метод/перегрузка методов|перегрузка методов]]
 - методы могут быть рекурсивными - вызывать сами себя
```
	static int factorial(int x){
		if (x == 1){
			return 1;
		}
		return x * factorial(x-1);
	    }
```
 - существуют абстрактные методы, их определение происходит с помощью модификатора [[пакет/класс/модификаторы/abstract|abstract]]
	абстрактный метод необходим только с целью его переопределения в производном классе. может не иметь блока кода
 - существуют методы "по умолчанию, их определение происходит с помощью модификатора [[пакет/класс/модификаторы/default|default]]"
	в отличие от обычных методов, разрешенных в интерфейсах эти могут иметь внутри себя [[блок кода]], но могут быть так же переопределены в других классах

Вызов метода может происходить двумя способами:
- С помощью знака `.` -  `Class.method()` - наиболее частый вариант, используемый в любом месте код
- С помощью [[java/пакет/класс/метод/Ссылки на методы|ссылки]] - `Class::method` - передает вместо [[java/управляющие конструкции/Лямбда-выражения|лямбда-выражения]] указатель на метод, позволяя сократить код.
