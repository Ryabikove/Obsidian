способ определения методов, при котором производный класс определяет метод, название которого идентично названию метода из родительского класса

пример:
public class Program{
	public static void main(String\[] args) {
	}
}
class Class1 extends Class2 {
	@Override
	public void method1(){
		actions
	}
}
class Class2{
	public void method1(){
		actions
	}
}

==требования==:
 - переопределенный метод в производном классе должен иметь уровень доступа не меньше, чем уровень у метода в базовом классе


==особенности==:
 - переопределенный метод так же доступен, но только внутри производного класса через ключевое слово [[java/пакет/класс/наследование/super|super]]
 - перед переопределением метода указывается аннотация @Override, она не обязательна, но дает понять, что метод переопределен
 - можно ==запретить переопределение методов== ключевым словом [[пакет/класс/модификаторы/final|final]]
	modificators... final method1(){}
