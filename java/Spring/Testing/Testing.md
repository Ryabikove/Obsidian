There is main conception of testing "Testing Pyramid". The idea is to have the more and the simplier tests for the lower parts of code and the complexier and the lesser tests for hight and complext parts of code.
There is 6 levels of tests (from the most simple to the most complex):
1. [[Spring/Testing/Unit tests/Unit tests|Unit tests]] - testing single units in total isolation.
2. Component tests
3. Integration tests
4. API tests
5. UI tests
6. Manual tests
