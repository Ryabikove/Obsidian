[[Study/1 java core/9 Stream-api|9 Stream-api]] поддерживает параллелизацию, таким образом эл-ты потока обрабатываются параллельно при задействовании сразу нескольких ядер процессора.

Параллельные потоки используют фреймворк Fork/Join:
1. Split (разделение) - источник данных разделяется на части;
2. Execute (выполнение) - каждая часть обрабатывается в отдельном потоке;
3. Combine (объединение) - результат всех потоков склеивается обратно в один;

Способы вызова параллельных потоков:
- Для [[Study/1 java core/5 Collections|коллекции]] через метод `parallelStream()` - `planets.parallelStream().filter(p -> p.length() > 5).toList();`
- Для [[Java/variables/атрибуты переменных/значения переменной/Array|массива]] через метод `parallel()` класса-коллекции - `Arrays.stream(arr).parallel().filter(p -> p.length() > 5).toList();`

==Преимущества:==
- Высокая скорость обработки большого объема данных в нескольких потоках;

==Недостатки==
- Непредсказуемые результаты при изменении внешних(не потокобезопасных) переменных;
- При работе с небольшими объемами данных накладные работы на разделение и склейку задач могут занять больше времени, чем сама обработка;
- Зависимость от порядка - операции вроде `findFirst()` или `limit()` в параллельном режиме работают медленнее или сложнее;
- При работе с многомерными массивами нагрузка может разделиться неравномерно;
- При работе с IO операциями (чтение файлов, запросы в БД) может заблокироваться CommonPool из-за ожидания ответа от сети/диска;
- Медленная работа с [[Java/package/built-in/java/util/LinkedList]] из-за трудности в разделении на равные части;

Для определения ==рациональности== использования многопоточности можно применить правило: **N** (количества элементов) \* **Q** (сложность вычислений на один элемент)  > 10 000.
