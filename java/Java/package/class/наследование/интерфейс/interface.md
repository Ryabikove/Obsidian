конструкция, определяющая нереализованный функционал, не имеющий конкретной реализации, который реализуют классы

одним из самых распространенных способов использования интерфейсов является создание обратного вызова (действие, вызываемое при других действиях)

синтаксис:
 interface Printable{
	 void print(); 
 }

==особенности==:
 - может содержать внутри себя поля(переменные) и методы
 - методы и поля могут быть как реализованными, так и "абстрактными" (не обязательно добавлять модификатор)
 - поля и методы 
 - объектам интерфейсов можно передавать ссылки на классы, которые их применяют
	Int1 a = new Class1() 
 - [[Java/variables/attributes/data type/type convertion/Type conversion|Type conversion]] для интерфейсов применяется так же, как и для классов
 - принцип наследования и вложенности у интерфейсов точно такой же, как и у классов
	[[Java/package/class/наследование/Наследование|Наследование]] [[Java/package/class/Inner classes|Inner classes]]
 - интерфейсы могут использоваться как и классы в качестве параметров метода или возвращаемого типа
	interface int1{}
	class Class1 implements int1{
		modificators int1 method1(int1 a){
			actions;
		}
	}
 - при написании интерфейса можно задать универсальный параметр, позволяющий определять тип полей интерфейса при создании экземпляра - обобщенный интерфейс
	[[Java/package/class/обобщения/generics-обобщения|generics-обобщения]]
 - существуют встроенные интерфейсы

==требования==:
 - поля(переменные) по умолчанию имеют модификаторы public static final, поэтому обязательно должны быть определены. приватные поля запрещены в интерфейсах, за исключением переопределенного интерфейса, передаваемого экземпляру класса в конструктор.
	![[Java/Pasted image 20241104205929.png]]
 - методы без модификаторов по умолчанию абстрактные и не могут иметь [[Java/constructions/Code block|Code block]]
 - приватные методы интерфейса можно использовать только внутри самого интерфейса, и они не могут иметь базовой реализации ([[Java/variables/modificators/default|default]])
 - если класс не реализует все методы, то такой класс должен быть помечен как абстрактный
 - для того, чтобы метод имел блок кода, ему необходимо добавить модификатор [[Java/variables/modificators/default|default]] или он должен быть статичным [[Java/variables/modificators/static|static]]
 - нельзя напрямую создавать объекты интерфейсов
	Int1 a = new Int1() \# вызовет ошибку
 - интерфейс может быть определен внутри другого класса или интерфейса
