способ определения класса, интерфейса или метода, позволяющий определить тип данных принимаемых параметров при создании экземпляра класса или наследовании интерфейса/класса

универсальный параметр - параметр, заключенный в угловые скобки \<\>
имеет вид
\<Str\>

==особенности==:
 - при объявлении обобщенного метода универсальный параметр указывается после всех модификаторов и перед типом возвращаемого значения
 - тип данных у обобщенного метода может различаться при каждом вызове
 - можно использовать несколько универсальных параметров, они перечисляются через запятую
	\<R,S\>
 - конструкторы, как и другие методы могут быть обобщенными
 - можно ограничить набор типов, которые могут быть указаны при объявлении/вызове
	в этом случае используется ключевое слово [[Java/package/class/наследование/extends|extends]], после которого указывается базовый класс
	только этот или дочерние классы могут быть указаны в качестве типа данных
	\<A extends B\>
 - при ограничении типов внутри обобщенного класса можно использовать методы и поля класса-ограничителя, даже если они определяются в передающемся классе, а не ограничивающем
 - в качестве ограничений могут выступать другие обобщения или интерфейсы
	\<A extends B\<C\>\> \<A extends Int\>
 - ограничения могут быть так же множественными
	class Person{}; interface Accountable{}; \<T extends Person & Accountable\>
 - указание типов обобщенного класса при наследовании может быть не всегда от прямого наследника

==требования==:
 - в угловых скобках при объявлении класса/интерфейса указывается любая произвольная строка длиной от 1 символа
 - при создании экземпляра или наследовании класса/интерфейса внутри угловых скобок может быть только объект (Integer, String, Double...). указание примитивного типа (int,double...) приведет к ошибке
