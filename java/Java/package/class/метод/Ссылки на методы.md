Способ вызова метода, используемый вместо [[Java/constructions/управляющие конструкции/Лямбда-выражения|лямбда-выражений]].
Используется в случаях, когда лямбда выражение вызывает метод без дополнительной логики. 

Синтаксис:
```
s -> s.method() # было
Class::method # стало
```


Ссылки бывают 4 типов:
- Статические - `Class::method` - используется, когда требуется вызвать статический метод класса
```
int a = 10;
UnaryOperator<Integer> sqrt = Math::sqrt;
a = sqrt.apply(a);
```
- Объектов - `object::method` - используется, когда требуется вызвать метод конкретного объекта 
```
String a = "Object means: ";
UnaryOperator<String> adder = a::concat;
```
- Произвольного объекта - `Class::method` - используется при вызове метода произвольного объекта, переданного в лямбду первым аргументом
```
String a = "qwerty";
UnartOperator<String> toUpper = String::toUpperCase;
a = toUpper.apply(a);
```
- Конструктор - `Class::new`


Использование ссылок невозможно если:
- Помимо вызова метода используется доп. логика: 
	`s -> Class.method(s) + 12`
- Метод принимает несколько аргументов, но необходимо поменять их местами или один сделать константой:
	`s -> s.method(0,3) # 0 и 3 константы, зашитые внутрь`
	`(s,n) -> s.method(n) # такое можно, т.к. то же, что и Class::method`
- У класса есть несколько перегруженных методов с одинаковым именем, т.к. компилятор не поймет, какой именно имеется в виду.
