Project design - is the creation of a foundation on which the code and business logic will stand.

==Fixing a bug at this stage will cost tens of times less than fixing it at the coding stage.==

Design stages:
1. Research:
	1. [[Autogeek/1 Research/Analyse of market and competitors|Analyse of market and competitors]] - examine the similar products;
	2. [[Autogeek/1 Research/Target client|Create an image of target client]];
	3. [[Autogeek/1 Research/Target client|Problems that we solve]] - visualization of the user's path from the emergence of a problem to its solution;
2. Requirements collection:
	1. [[Autogeek/2 Requirements collection/Functional requirements|Functional requirements]] - what the product should do;
	2. [[Autogeek/2 Requirements collection/Non-functional requirements|Non-functional requirements]] - how the product should work (safety, load, response speed);
	3. [[Autogeek/2 Requirements collection/User Stories|User Stories]] - short descriptions of functions in the format "I, as a user, want \[action] to \[profit]".
3. Data architecture - data structure and relates between entities.
	1. [[Autogeek/3 Data architecture/Maps/Map|Map of site or app]] - hierarchy of modules
	2. [[Autogeek/3 Data architecture/Maps/Map.canvas|Logical schema (Flowcharts)]] - representation of connections between modules
4. UX-projecting (Wireframing) - creationg of raw modules without design patterns
	1. Low-fidelity - raw notes of logical connections
	2. Connection testing - testing the intuitiveness of the interface
5. UI-design (Visual Design) - creation of attractive interface
	1. Design System - bunch of entities which makes modules look with same design
	2. High-fidelity prototype - prototype of completed product
6. Technical projecting
	1. Making the technology stack - programming languages, DB, frameworks
	2. Projecting DB
	3. API
