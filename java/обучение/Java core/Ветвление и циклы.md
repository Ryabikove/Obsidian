В любом программном коде присутствуют ветвления и циклы, т.е. те части кода, которые в зависимости от текущего состояния объектов выполняют те или иные действия.

Основные циклы:
- [[java/управляющие конструкции/циклы/for|for]] (*элементы условия*) {код} - способ итерации, который проходит по всем элементам условия, условие проверяется сначала
- [[java/управляющие конструкции/циклы/while|while]] (*условие*) {код} - способ итерации, который выполняет код до тех пор, пока условие истинно (true), условие проверяется сначала
- do {код} [[java/управляющие конструкции/циклы/while|while]] (условие) - способ итерации, в котором сначала выполняется код, а потом проверяется условие

Ветвления:
- [[java/управляющие конструкции/условные/if else|if (условие){код} else{код}/else if(условие) {код}]] - классический оператор ветвления, принимает условие, в зависимости от результата выполняет код в блоке if или else
- [[java/управляющие конструкции/условные/switch|switch]] - оператор ветвления, который принимает объект, выражение или реже условие, и сравнивает результат со значениями в блоках case, при обнаружении первого совпадения выполняет код. ==Из-за особенностей работы switch в блоках case размещают оператор break, иначе при нахождении первого совпадения оператор будет выполнять код не только в первом совпадении, но и во всех последующих, даже если в них условия неверны.==


Так же существуют ключевые слова для управления поведением внутри циклов:
- [[java/управляющие конструкции/циклы/break|break]] - при вызове прерывает выполнение текущего цикла.
- [[java/управляющие конструкции/циклы/continue|continue]] - при прерывает выполнение кода в теле цикла и переходит к следующей итерации

Для операторов break и continue есть возможность выставления [[java/управляющие конструкции/циклы/метки|меток]], которые позволяют влиять на выполнение не только текущего цикла, но и внешних. 


==Условие должно возвращать значения только типа boolean.==
==В отличие от других языков, преобразование других типов в тип boolean автоматически не происходит: 0,  пустая строка, пустая переменная, null не преобразуются в false. Тот же принцип работы и с true.==
