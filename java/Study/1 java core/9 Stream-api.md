[[Java/constructions/управляющие конструкции/Stream-api/Stream-api|Stream-api]] - набор инструментов, предназначенный для удобной эффективной обработки последовательностей элементов в функциональном стиле.

Ключевая концепция - stream - конвейер операций над элементами, взятыми из источника.

Основные св-ва:
- Данные принимаются из источника и после обработки передаются сразу дальше, а не хранятся в памяти;
- Операции возвращают новый поток или результат, но не модифицируют исходные данные;
- Промежуточные операции не выполняются сразу. Реальные вычисления запускаются только при вызове терминальной операции;
- Потоки могут быть последовательными или параллельными;

Основные ==преимущества==:
- Функциональный стиль - поддержка [[Java/constructions/управляющие конструкции/Лямбда-выражения|лямбда-выражений]] и [[Java/package/class/метод/Ссылки на методы|ссылок на методы]];
- Параллелизация - возможность переключения на параллельную обработку;
- Гибкость - можно комбинировать операции в цепочки для сложной обработки данных;

==Недостатки==:
- Одноразовость - после терминальной операции нельзя использовать повторно;
- Ограниченная область применения - не все задачи удобно решать с помощью потоков;
