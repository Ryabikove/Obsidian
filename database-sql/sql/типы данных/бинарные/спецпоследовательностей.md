формат кодирования бинарного типа данных
двоичная строка представляется в виде последовательности ASCII-символов, непредставимые байты передаются в виде спецпоследовательностей

==традиционно использовался в PostgreSQL для значений типа bytea==

==может быть удобен, если с точки зрения приложения представление байт в виде символов имеет смысл==

==на практике это обычно создаёт путаницу, так как двоичные и символьные строки могут выглядеть одинаково==

Передавая значения в формате спецпоследовательности, байты с определёнными значениями ==необходимо== записывать специальным образом, хотя так можно записывать и все значения.
для этого значение байта нужно:
 1. преобразовать в трёхзначное восьмеричное число
 2. добавить перед ним обратную косую черту
	саму обратную косую черту можно записать в виде двух таких символов
