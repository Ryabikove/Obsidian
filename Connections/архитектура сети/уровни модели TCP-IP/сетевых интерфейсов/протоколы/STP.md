Spanning Tree Protocol - протокол связующего дерева, позволяет автоматически отключать дублирующие соединения между коммутаторами и включать их при необходимости

связующее дерево содержит подграф без циклов, содержащий все вершины(корневые уст-ва) исходного графа

работа протокола делится на несколько этапов:
 1.  выбор корневого уст-ва
    автоматически выбирается минимальный mac адрес, либо же можно выбрать вручную
 2.  определение кратчайших путей до корневого уст-ва согласно наименьшей стоимости
	основными показателями для определения кратчайшего пути выступают: 
	- количество промежуточных уст-в
	- скорость соединения 
	все уст-ва рассылают информацию о кратчайшем расстоянии от них до корневого уст-ва
 3.  отключение всех соединений, кроме кратчайших
	если найдено несколько соединений, имеющих одинаковую стоимость, отключаются все, кроме наименьшего по значению порта

выбор корневого уст-ва происходит по следующему алгоритму:
 1. каждое уст-во рассылает сообщения другим о том, что оно корневой
 2. получив сообщения, уст-во сравнивают свой идентификатор с соседними
 3. после чего снова рассылаются сообщения с минимальным идентификатором среди всех соседних
 4. когда в получаемых сообщениях идентификатор совпадает с отправляемым, рассылка прекращается

для реализации работы протокола STP коммутаторы обмениваются сообщениями BPDU (Bridge Protocol Data Units) каждые 2 секунды
для адреса получателя выбирается групповой mac адрес STP - 01:80:C2:00:00:00
