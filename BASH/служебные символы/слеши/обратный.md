"escape"
выполняет [[строгое]] [[экранирование]] одного символа в строке, перед которым стоит, эффект будто символ заключен в [[одинарные]] кавычки

 может так же экранировать сами кавычки

некоторые обычные символы приобретают специальные св-ва, когда перед ними стоит "escape" символ [[специальные экранированные символы]]

поведение символа \\ зависит от того экранирован он, ограничен кавычками, находится внутри конструкции подстановки команды или во вложенном документе.
 echo \\z          \#  z
 echo \\\\z         \# \\z
 echo '\\z'         \# \\z
 echo '\\\\z'        \# \\\\z
 echo "\\z"         \# \\z
 echo "\\\\z"        \# \\z
 echo \`echo \\z\`     #  z
 echo \`echo \\\\z\`    #  z
 echo \`echo \\\\\\z\`      # \\z
 echo \`echo \\\\\\\\z\`     # \\z
 echo \`echo \\\\\\\\\\\\z\`   # \\z
 echo \`echo \\\\\\\\\\\\\\z\`  # \\\\z
 echo \`echo "\\z"\`      # \\z
 echo \`echo "\\\\z"\`     # \\z

позволяет экранировать символ перевода строки, в результате чего одна команда может занимать несколько строк
 df -h | \
 awk 'print$3' >/dev/null
 echo "foo
 bar" \# двустрочный вывод
 echo 'foo
 bar'    # двустрочный вывод
 echo foo\
 bar     # однострочный вывод
 echo "foo\
 bar"     # однострочный вывод
 echo 'foo\
 bar'     # двустрочный вывод (\\ экранирован)

 ==Если строка сценария заканчивается символом [[прямой]] слеш, то необходимость в применении символа \ отпадает==
