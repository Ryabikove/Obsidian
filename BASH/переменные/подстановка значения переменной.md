когда [[интерпретатор]] встречает в тексте сценария имя [[переменная]], вместо имени он вставляет значение этой переменной. Поэтому ссылки на переменные называются подстановкой переменных.

[[знак доллар]] позволяет из имени переменной сделать ссылку
 var1=1
 echo $var1 \#1

 необходимо помнить о различиях между именем переменной и ее значением. 
 **var1** - имя переменной, **$var1** - ссылка на ее значение.

чистые имена переменных без [[знак доллар]] могут использоваться только при [[объявление переменной]], [[присваивание значения переменной]], [[удаление переменной]], [[экспорт переменной]] и в редких случаях, когда переменная представляет собой название сигнала

заключение ссылки на переменную в [[двойные]] кавычки никак не сказывается на работе механизма подстановки. Т.к. применяется [[нестрогое]] [[экранирование]]
 hello="A B  C   D"
 echo $hello   # A B C D
 echo "$hello" # A B  C   D # Заключение ссылки на переменную в кавычки сохраняет пробельные символы.

[[одинарные]] кавычки заставляют интерпретатор воспринимать ссылку на переменную как простой набор символов. Т.к. применяется [[строгое]] [[экранирование]].

написание **$var** фактически является упрощенной формой написания **\${var}**. 
 ==более строгая форма записи **\${var}** может с успехом использоваться в тех случаях, когда применение упрощенной формы записи порождает сообщения о синтаксических ошибках==
