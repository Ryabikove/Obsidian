что? основное понятие и объект, содержащий определенную информацию

зачем? необходимо для хранения, редактирования и использования пользователем информации, содержащейся в объекте 

особенности:
 1. запись имеет несколько базовых полей-атрибутов , не зависящих от типа:
  - [[базовые атрибуты/идентификатор|идентификатор]]
  - [[базовые атрибуты/оглавление|оглавление]]
  - [[базовые атрибуты/содержание|содержание]]
  - [[базовые атрибуты/время создания|время создания]]
  - [[базовые атрибуты/время последнего изменения|время последнего изменения]]
  - [[базовые атрибуты/абсолютная дата|абсолютная дата]]
 2. запись может быть одновременно только одним из нескольких типов:
  - [[тип записи/ежедневная/ежедневная|ежедневная]]
  - [[тип записи/расход/расход|расход]]
  - [[доход]]
  - [[тип записи/цель/цель|цель]]
 3. все записи при создании являются самостоятельными и не ссылаются автоматически на другие, если шаблоном или условиями создания не задано обратного
 4. каждая запись может ссылаться на другие записи как индивидуально ([[группирование/ссылка|ссылка]]), так и на группы в целом ([[тег]]), вне зависимости от типа записи
 5. записи типа [[цель]], [[тип записи/расход/расход|расход]] и [[доход]] имеют строгую горизонтальную иерархию:
  - могут быть двух подтипов [[тип записи/главная|главная]] и [[тип записи/второстепенная|второстепенная]].
  - у каждой записи типа второстепенная может быть только одна запись типа главная
  - главная запись не может стать второстепенной для записи стоящей ниже по иерархии
  - у главной записи может быть множество второстепенных
  - запись может быть одновременно главной для множества других и второстепенной только для одной 
 6. при отображении в пользовательском интерфейсе записи собираются по шаблону
